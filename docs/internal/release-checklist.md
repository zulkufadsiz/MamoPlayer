# Release Checklist

Use this checklist before each release of `@mamoplayer/core` and/or `@mamoplayer/pro`.

---

## 1. Builds

- [ ] `yarn build:core` completes without errors
- [ ] `yarn build:pro` completes without errors
- [ ] No TypeScript errors in either package (`tsc --noEmit` passes)

## 2. Tests

- [ ] `yarn test:packages` passes (core and pro test suites)
- [ ] No skipped or `.only` tests left in committed code

## 3. Example App

- [ ] iOS example app runs correctly (`yarn start:example`)
- [ ] Android example app runs correctly (`yarn start:example:android`)
- [ ] Core features demonstrated in `CoreDemoScreen` work as expected
- [ ] Pro features demonstrated in `ProDemoScreen` work as expected

## 4. Docs Site

- [ ] `yarn docs:build` completes without errors
- [ ] Changelog (`docs/docs/changelog.md`) is updated for this release
- [ ] Any new APIs or breaking changes are documented

## 5. Import Paths

- [ ] All consumer-facing imports reference `@mamoplayer/core` (not relative root paths)
- [ ] All consumer-facing imports reference `@mamoplayer/pro` (not relative root paths)
- [ ] No internal cross-package imports bypass the published package boundaries

## 6. Legacy / Root Files

- [ ] No old root-level source files remain that should have been moved to `packages/`
- [ ] `index.js` at root is intentional (entry for RN app) — not a leftover package entry
- [ ] `trackPlayerService.ts` and `trackPlayerSetup.ts` at root are intentional — not duplicates of package internals

## 7. Core Package — Public Publish

- [ ] `packages/core/package.json` version is correct and follows semver
- [ ] `publishConfig.access` is set to `"public"`
- [ ] `main`, `module`, and `types` fields point to built output in `dist/`
- [ ] `files` field includes only what should be shipped (e.g. `dist/`, `README.md`)
- [ ] Peer dependencies (`react`, `react-native`) are correct and not pinned too tightly
- [ ] CHANGELOG entry exists for this version

## 8. Pro Package — Private Publish

- [ ] `packages/pro/package.json` version is correct and follows semver
- [ ] `publishConfig.access` is set to `"restricted"` (npm) or omitted for GitHub Packages
- [ ] `main`, `module`, and `types` fields point to built output in `dist/`
- [ ] `files` field includes only what should be shipped
- [ ] Registry target is confirmed (npm private org or GitHub Packages) — see `pro-distribution.md`
- [ ] No sensitive internal paths or tokens are referenced in published files

## 9. README & Install Examples

- [ ] Root `README.md` install snippet uses `@mamoplayer/core`
- [ ] `packages/core/README.md` install snippet is accurate and up to date
- [ ] `packages/pro/README.md` install snippet reflects the correct registry and auth setup
- [ ] Any `.npmrc` examples in docs match the currently supported registries

## 10. Version Alignment

- [ ] Lerna versioning strategy confirmed (`lerna.json` uses `"version": "independent"`)
- [ ] If both packages are released together, their peer/dependency declarations reference compatible version ranges
- [ ] Git tag(s) created by `lerna version` match the published versions
- [ ] `CHANGELOG.md` entries generated by conventional commits are accurate

---

## Pre-publish Commands

```bash
# Full build
yarn build

# Full test suite
yarn test:packages

# Lint and format check
npm run lint
npm run format:check

# Docs build
yarn docs:build

# Dry-run release (semantic-release)
npm run release:dry
```

## Publish Commands

```bash
# Core — public
cd packages/core
npm publish --access public

# Pro — private (npm)
cd packages/pro
npm publish --access restricted

# Pro — private (GitHub Packages)
cd packages/pro
npm publish --registry=https://npm.pkg.github.com
```
